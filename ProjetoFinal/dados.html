Banco de Dados

Como as informações dos usuários são organizadas.

Estrutura dos Dados

O banco de dados armazena informações essenciais como usuários, músicas, artistas, playlists, curtidas e histórico de reprodução. Cada usuário possui um identificador único que se relaciona com suas preferências musicais.

Com esses dados bem organizados, o sistema consegue buscar rapidamente informações para gerar recomendações, playlists automáticas e estatísticas personalizadas para cada perfil de usuário.

Modelo do Banco de Dados

Tabela Usuários: armazena os dados dos usuários como id, nome, e-mail e senha.

Tabela Músicas: guarda as informações das músicas como id, nome, artista e gênero.

Tabela Playlists: armazena as playlists criadas pelo usuário.

Essas tabelas se relacionam entre si por meio de chaves estrangeiras, permitindo que o sistema saiba quais músicas cada usuário ouve, curte e recebe como recomendação.

Relacionamentos

Um usuário pode criar várias playlists.

Uma playlist pode conter várias músicas.

As músicas podem aparecer em várias playlists diferentes.

Essas três tabelas são suficientes para o funcionamento da playlist algorítmica, pois o sistema precisa saber quem é o usuário, quais músicas existem e quais playlists estão sendo geradas automaticamente.
